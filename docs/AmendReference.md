## 概述

修约功能用于对单元格中的数值进行精度处理，支持多种修约方式和精度设置。修约设置包含以下四个参数：

| 参数 | 说明 |
|------|------|
| amendType | 修约方式代码 |
| presion | 修约精度 |
| displayValue | 显示格式 |
| fixnum | 固定有效数字位数（仅修约方式为"11"时有效） |

---

## 修约方式

### 支持的修约方式

| 代码 | 名称 | 说明 |
|------|------|------|
| `00` | 四舍六入五成双 | GB/T 8170 标准修约规则，也称"银行家舍入" |
| `22` | 四舍五入 | 传统四舍五入规则 |
| `05` | 零五修约 | 修约到0.5的倍数 |
| `03` | 零三修约 | 修约到0.3的倍数 |
| `02` | 零二修约 | 修约到0.2的倍数 |
| `04` | 非零进位修约 | 任何非零数字都进位 |
| `11` | 固定有效数字位数 | 按有效数字位数修约 |

---

### 修约方式详解

#### 1. 四舍六入五成双（代码：00）

这是 GB/T 8170《数值修约规则与极限数值的表示和判定》中规定的标准修约规则，也称"银行家舍入"。

**规则说明：**
- 当尾数小于5时，直接舍去
- 当尾数大于5时，进位
- 当尾数等于5时：
  - 如果5后面还有非零数字，则进位
  - 如果5后面没有数字或全是0，则看5前面的数字：
    - 前一位是奇数，则进位
    - 前一位是偶数，则舍去

**示例（修约到小数点后一位）：**
```
1.14 → 1.1   （尾数4 < 5，舍去）
1.16 → 1.2   （尾数6 > 5，进位）
1.15 → 1.2   （尾数5，前一位1是奇数，进位）
1.25 → 1.2   （尾数5，前一位2是偶数，舍去）
1.151 → 1.2  （尾数5后还有非零数字，进位）
```

---

#### 2. 四舍五入（代码：22）

传统的四舍五入规则，简单直观。

**规则说明：**
- 当尾数小于5时，直接舍去
- 当尾数大于等于5时，进位

**示例（修约到小数点后一位）：**
```
1.14 → 1.1   （尾数4 < 5，舍去）
1.15 → 1.2   （尾数5 ≥ 5，进位）
1.16 → 1.2   （尾数6 ≥ 5，进位）
```

---

#### 3. 零五修约（代码：05）

修约结果为0.5的倍数，常用于某些工程测量领域。

**规则说明：**
- 将原数值乘以2
- 按四舍六入五成双规则修约到指定位数
- 将结果除以2

**示例（修约到个位）：**
```
1.2 → 1.0    （1.2×2=2.4→2.0→1.0）
1.3 → 1.5    （1.3×2=2.6→3.0→1.5）
1.7 → 1.5    （1.7×2=3.4→3.0→1.5）
1.8 → 2.0    （1.8×2=3.6→4.0→2.0）
```

---

#### 4. 零三修约（代码：03）

修约结果为0.3的倍数。

**规则说明：**
- 将原数值乘以10/3
- 按四舍六入五成双规则修约
- 将结果乘以0.3

**示例（修约到个位）：**
```
0.9 → 0.9    （0.9×10/3=3.0→3.0×0.3=0.9）
1.2 → 1.2    （1.2×10/3=4.0→4.0×0.3=1.2）
1.5 → 1.5    （1.5×10/3=5.0→5.0×0.3=1.5）
```

---

#### 5. 零二修约（代码：02）

修约结果为0.2的倍数。

**规则说明：**
- 将原数值乘以5
- 按四舍六入五成双规则修约
- 将结果除以5

**示例（修约到个位）：**
```
1.1 → 1.0    （1.1×5=5.5→5.0→1.0）
1.3 → 1.2    （1.3×5=6.5→6.0→1.2）
1.5 → 1.6    （1.5×5=7.5→8.0→1.6）
```

---

#### 6. 非零进位修约（代码：04）

任何非零数字都进位，常用于安全系数计算等场景。

**规则说明：**
- 只要尾数不为0，就进位
- 尾数为0时，舍去

**示例（修约到小数点后一位）：**
```
1.10 → 1.1   （尾数0，舍去）
1.11 → 1.2   （尾数1 ≠ 0，进位）
1.19 → 1.2   （尾数9 ≠ 0，进位）
```

**示例（修约到个位）：**
```
10.0 → 10    （尾数0，舍去）
10.1 → 11    （尾数1 ≠ 0，进位）
10.9 → 11    （尾数9 ≠ 0，进位）
```

---

#### 7. 固定有效数字位数（代码：11）

按有效数字位数进行修约，需配合 fixnum 参数使用。

**规则说明：**
- 保留指定位数的有效数字
- 使用科学计数法表示大数或小数

**示例（保留3位有效数字）：**
```
1234 → 1.23E3
12.34 → 12.3
1.234 → 1.23
0.01234 → 1.23E-2
```

## 修约精度

修约精度决定了修约到哪一位。

### 精度代码对照表

| 代码 | 说明 | 示例 |
|------|------|------|
| `-2` | 小数点前三位（百位） | 1234 → 1200 |
| `-1` | 小数点前两位（十位） | 123 → 120 |
| `0` | 小数点前一位（个位） | 12.3 → 12 |
| `1` | 小数点后一位 | 1.23 → 1.2 |
| `2` | 小数点后二位 | 1.234 → 1.23 |
| `3` | 小数点后三位 | 1.2345 → 1.235 |
| `4` | 小数点后四位 | 1.23456 → 1.2346 |
| `5` | 小数点后五位 | 1.234567 → 1.23457 |
| `6` | 小数点后六位 | 1.2345678 → 1.234568 |
| `7` | 小数点后七位 | 1.23456789 → 1.2345679 |

## 显示格式

显示格式用于控制数值的显示方式，不影响实际存储值。

### 普通格式

| 格式 | 说明 | 示例 |
|------|------|------|
| `0` | 整数 | 123.45 → 123 |
| `0.0` | 一位小数 | 123.456 → 123.5 |
| `0.00` | 二位小数 | 123.4567 → 123.46 |
| `0.000` | 三位小数 | 123.45678 → 123.457 |
| `0.0000` | 四位小数 | 123.456789 → 123.4568 |
| `0.00000` | 五位小数 | - |
| `0.000000` | 六位小数 | - |
| `0.0000000` | 七位小数 | - |

### 科学计数法格式

| 格式 | 说明 | 示例 |
|------|------|------|
| `0.0E+00` | 科学计数法（1位小数） | 1234.5 → 1.2E+03 |
| `0.00E+00` | 科学计数法（2位小数） | 1234.56 → 1.23E+03 |
| `0.000E+00` | 科学计数法（3位小数） | 1234.567 → 1.235E+03 |
| `0.0000E+00` | 科学计数法（4位小数） | - |
| `0.00000E+00` | 科学计数法（5位小数） | - |
| `0.000000E+00` | 科学计数法（6位小数） | - |
| `0.0000000E+00` | 科学计数法（7位小数） | - |

## 修约示例

### 示例1：四舍六入五成双修约

```javascript
// 设置修约参数
operCell.setAmend({
    amendType: "00",      // 四舍六入五成双
    presion: "2",         // 小数点后二位
    displayValue: "0.00"  // 显示格式
});

// 数值修约结果
// 1.234 → 1.23
// 1.235 → 1.24（前一位3是奇数，进位）
// 1.245 → 1.24（前一位4是偶数，舍去）
// 1.246 → 1.25
```

### 示例2：四舍五入修约

```javascript
// 设置修约参数
operCell.setAmend({
    amendType: "22",      // 四舍五入
    presion: "1",         // 小数点后一位
    displayValue: "0.0"   // 显示格式
});

// 数值修约结果
// 1.24 → 1.2
// 1.25 → 1.3
// 1.26 → 1.3
```

### 示例3：零五修约

```javascript
// 设置修约参数
operCell.setAmend({
    amendType: "05",      // 零五修约
    presion: "0",         // 个位
    displayValue: "0.0"   // 显示格式
});

// 数值修约结果
// 1.2 → 1.0
// 1.3 → 1.5
// 1.7 → 1.5
// 1.8 → 2.0
```

### 示例4：非零进位修约

```javascript
// 设置修约参数
operCell.setAmend({
    amendType: "04",      // 非零进位修约
    presion: "0",         // 个位
    displayValue: "0"     // 显示格式
});

// 数值修约结果
// 10.0 → 10
// 10.1 → 11
// 10.9 → 11
```

### 示例5：固定有效数字位数

```javascript
// 设置修约参数
operCell.setAmend({
    amendType: "11",      // 固定有效数字位数
    fixnum: "3",          // 3位有效数字
    displayValue: ""      // 不设置显示格式
});

// 数值修约结果
// 1234 → 1.23E3
// 12.34 → 12.3
// 1.234 → 1.23
// 0.01234 → 1.23E-2
```

### 示例6：科学计数法显示

```javascript
// 设置修约参数
operCell.setAmend({
    amendType: "00",          // 四舍六入五成双
    presion: "2",             // 小数点后二位
    displayValue: "0.00E+00"  // 科学计数法格式
});

// 数值显示结果
// 1234.56 → 1.23E+03
// 0.001234 → 1.23E-03
```

## API 使用

### 设置修约参数

通过 MyCellDesigner 的 setAmend 方法设置修约参数：

```javascript
myCellDesigner.setAmend({
    amendType: "00",      // 修约方式
    presion: "2",         // 修约精度
    displayValue: "0.00", // 显示格式
    fixnum: ""            // 有效数字位数（仅amendType为"11"时使用）
});
```

### 编程方式修约

使用 Amend 类进行数值修约：

```javascript
var amend = new Amend();

// 综合修约方法
var result = amend.gf_amel_all(
    value,          // 数值
    amendType,      // 修约类型
    presion,        // 精度
    displayValue,   // 显示格式
    fixnum          // 有效位数
);

// 设置显示格式
var formattedValue = amend.setFormat(value, {
    amendType: "00",
    presion: "2",
    displayValue: "0.00",
    fixnum: ""
});
```

### 修约函数对照表

| 函数名 | 修约方式 | 说明 |
|--------|----------|------|
| `gf_amel(value, precision)` | 四舍六入五成双 | 标准修约函数 |
| `gf_amel45(value, precision)` | 四舍五入 | 传统四舍五入 |
| `gf_amelhalf(value, precision)` | 零五修约 | 修约到0.5的倍数 |
| `gf_amel02(value, precision)` | 零二修约 | 修约到0.2的倍数 |
| `gf_amel03(value, precision)` | 零三修约 | 修约到0.3的倍数 |
| `gf_amelZero(value, precision)` | 非零进位修约 | 非零即进位 |
| `gf_dotEx(value, n)` | 有效位数修约 | 保留n位有效数字 |
| `gf_amel_all(value, amendType, presion, displayValue, fixnum)` | 综合修约 | 根据类型调用对应修约函数 |

## 注意事项

1. **修约精度与显示格式的区别**
   - 修约精度（presion）改变数值的实际值
   - 显示格式（displayValue）只改变显示方式，不改变实际值

2. **修约方式的选择**
   - 一般科学计算推荐使用"四舍六入五成双"（代码00）
   - 商业计算常用"四舍五入"（代码22）
   - 工程测量根据行业规范选择

3. **有效数字位数修约**
   - 修约方式为"11"时，fixnum 参数才生效
   - 结果可能以科学计数法显示

4. **空值处理**
   - 空值或非数字值不进行修约
   - 修约函数会返回原值或空字符串
